# Payment Extension — API Reference

## APIs You Implement (Fynd calls your extension)

### 1. POST initiatePaymentSession
Fynd calls this to start a payment transaction.

**Endpoint**: `{Extension_URL}/initiate-payment` (configure in Partners Panel)

**Request Body:**
| Field | Type | Description |
|---|---|---|
| `gid` | string | Global transaction ID (unique per payment, generated by Fynd) |
| `success_url` | string | Redirect here after successful payment |
| `cancel_url` | string | Redirect here after failed/cancelled payment |
| `amount` | integer | Amount in smallest currency unit (paise for INR, cents for USD) |
| `currency` | string | Currency code (e.g., `INR`, `USD`) |
| `merchant_id` | string | Merchant identifier |
| `application_id` | string | Sales channel ID |
| `company_id` | string | Seller's company ID |

**Your Response:**
| Field | Type | Description |
|---|---|---|
| `redirect_url` | string | URL of payment gateway's hosted payment page |

**Headers received:** `checksum` (HMAC-SHA256 of request body — must verify)

---

### 2. GET getPaymentStatus
Fynd cron polls this to check transaction status.

**Endpoint**: `{Extension_URL}/payment-status`

**Query params:** `gid` (transaction ID)

**Your Response:**
| Field | Type | Description |
|---|---|---|
| `gid` | string | Transaction ID |
| `status` | string | `started` \| `pending` \| `complete` \| `failed` |

---

### 3. POST initiateRefundSession
Fynd calls this to start a refund.

**Endpoint**: `{Extension_URL}/initiate-refund`

**Request Body:**
| Field | Type | Description |
|---|---|---|
| `gid` | string | Original payment transaction ID |
| `request_id` | string | Unique refund request ID (created by Fynd) |
| `amount` | integer | Refund amount in smallest currency unit |
| `currency` | string | Currency code |
| `reason` | string | Refund reason |

**Your Response:**
| Field | Type | Description |
|---|---|---|
| `gid` | string | Original payment transaction ID |
| `status` | string | Current refund status |
| `aggregator_payment_refund_details` | array | Array of refund detail objects |

**Headers received:** `checksum` (must verify)

---

### 4. GET getRefundStatus
Fynd cron polls this to check refund status.

**Endpoint**: `{Extension_URL}/refund-status`

**Query params:** `gid` (original payment transaction ID)

**Your Response:** Same structure as `initiateRefundSession` response.

---

### 5. GET getActivationReadiness
Fynd calls this to check if merchant has provided credentials.

**Endpoint**: `{Extension_URL}/activation-readiness` (configure in Partners Panel)

**Query params:** `application_id`, `company_id`

**Your Response:**
```json
{ "success": true }   // credentials collected and valid
{ "success": false }  // credentials not yet collected
```

When `success: true`, merchant can activate the extension (status changes from "Available for Activation" to "Inactive").

---

## APIs You Call (Your Extension calls Fynd Platform)

### 6. PUT updatePaymentSession
Your extension calls this to update Fynd about payment status.

**SDK:**
```js
await platformClient.payment.updatePaymentSession({
  gid: transaction_gid,
  body: {
    status: "complete",   // started | pending | complete | failed
    checksum: your_checksum,
  }
});
```

### 7. PUT updateRefundSession
Your extension calls this to update Fynd about refund status.

**SDK:**
```js
await platformClient.payment.updateRefundSession({
  gid: original_payment_gid,
  body: {
    status: "refund_done",  // one of 6 refund statuses
    checksum: your_checksum,
  }
});
```

SDK Docs: https://docs.fynd.com/partners/commerce/sdk/latest/platform/application/payment

## AggregatorPaymentRefundDetails Schema
```json
{
  "amount": 50000,          // integer, required
  "currency": "INR",        // string, required
  "request_id": "...",      // string, required (Fynd-generated refund request ID)
  "status": "refund_done"   // one of 10 payment/refund statuses
}
```
