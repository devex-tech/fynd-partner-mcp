# Payment Extension — Overview

## What is a Payment Extension?

A payment extension integrates a payment gateway or aggregator into Fynd Commerce. It enables merchants to accept payments through their preferred payment provider.

**Core responsibilities:**
1. Collect merchant's payment gateway credentials (during installation)
2. Initiate payment sessions when customers checkout
3. Track and report payment status to Fynd Platform
4. Handle refund initiation and status reporting

---

## Registration in Partners Panel

1. Partners Panel → Extensions → Create Extension
2. Select **Extension type: Payment**
3. Fill: Extension Name, Contact Email, Logo, Description
4. Set **Extension URL** (HTTPS — your hosted server or ngrok for dev)
5. Permissions: payment extension auto-gets `updatePaymentSession` + `updateRefundSession`. Add more if needed.
6. Distribution: Public or Private
7. Save → note **Extension API Key** and **Extension API Secret**

---

## Architecture

```
Customer checkout
      ↓
Storefront → Fynd Platform
      ↓
Fynd calls initiatePaymentSession → Your Extension
      ↓
Your Extension → Payment Gateway (creates session)
      ↓
Payment Gateway returns redirect_url
      ↓
Your Extension returns redirect_url → Fynd → Customer redirected
      ↓
Customer completes payment on Gateway's hosted page
      ↓
Gateway callback → Your Extension
      ↓
Your Extension calls updatePaymentSession (status: complete/failed)
      ↓
Fynd acknowledges → Customer redirected to success_url or cancel_url
```

---

## Key Identifiers

| Field | Description |
|---|---|
| `gid` | Global transaction ID — generated by Fynd, unique per payment |
| `company_id` | Seller's company ID |
| `application_id` | Sales channel ID — credentials stored per `company_id × application_id` |
| `Extension API Key` | Your extension identifier |
| `Extension API Secret` | Used for HMAC checksum generation/verification |

---

## Installation & Activation Flow

1. Seller installs extension → OAuth flow (same as general extension)
2. Extension collects merchant's **payment gateway credentials** (API key, secret, merchant ID from their gateway account)
3. Fynd calls `getActivationReadiness` → extension responds `true` if credentials collected, `false` if not
4. Status: `Available for Activation` → once credentials collected → `Inactive` → seller toggles on → `Active`

**Important**: Store credentials per `company_id × application_id` — a seller may have different gateway accounts for different sales channels.
