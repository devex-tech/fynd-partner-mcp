## Critical Facts (always keep in mind)

- **`gid`** = Global ID — unique transaction identifier generated by Fynd. Always use it to look up transactions.
- **Payment statuses**: `started` → `pending` → `complete` OR `failed` (4 states only for updatePaymentSession)
- **Refund statuses**: `refund_initiated` → `refund_pending` → `refund_done` OR `refund_failed` / `refund_rejected` / `refund_disputed`
- **Checksum** must be verified on incoming requests (`initiatePaymentSession`, `initiateRefundSession`) and generated for outgoing (`updatePaymentSession`, `updateRefundSession`)
- **Cron jobs**: Fynd calls `getPaymentStatus` and `getRefundStatus` periodically — always implement them
